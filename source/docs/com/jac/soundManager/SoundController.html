Packagecom.jac.soundManager
Classpublic class SoundController
InheritanceSoundController flash.events.EventDispatcher

A SoundController is intended to manipulate individual SMSounds or groups of SMSounds. Each SoundController contains a list of SMSound objects that can be controlled. It is recommended that all SMSound objects be controlled through a SoundController. However SMSound objects can function on their own in a limited capacity.

public function fadeAllVolume(endPercent:Number, duration:Number, curve:* = null):void

Fades the volume of all registered sound objects over time.

Calls allVolumeFadeStart(), when the fade begins.

Calls allVolumeFadeComplete(), when the fade ends.

Calls allVolumeFadeUpdate(), on every tween update.

Parameters
endPercent:Number — The final volume to fade to (0-1 range)
 
duration:Number — Length of time in seconds the fade should happen over.
 
curve:* (default = null) — A TweenLite/TweenMax curve def, to pass to the tween engine.

forceAllVolume()method 
public function forceAllVolume(vol:Number):void

This will force the volume of all registered sound objects Please note that this will override any other volume modifications currently applied to the objects. This will also stop any tweens from a fadeVolume() on this ISMSound before the volume is set.

Dispatches a SMSoundVolumeEvent.VOLUME_CHANGE event when called.

Chances are pretty good that you want to use setVolume in most cases.

Parameters
vol:Number — A volume to force all sounds to, 0-1. This number is not clamped at this level, however it may be clamped further down the chain.

See also

setAllVolume()

play()method 
public function play(soundID:String, volume:Number, startTimeInMS:Number = 0, loops:int):ISMSound

Plays the sound specified by the soundID. This will also kill any tweens/fades previously applied to this sound object

Dispatches a SMSoundEvent.PLAY event when called.

Parameters
soundID:String — the unique ID of the ISMSound compliant object.
 
volume:Number — the volume to play the sound at. This volume is limited by the SoundController's volume as well as the SoundManager's volume. If set to NO_VOLUME_CHANGE, the sounds current volume setting will be used.
 
startTimeInMS:Number (default = 0) — how many milliseconds into the sound to start from.
 
loops:int — the number of times to play the sound in a row. if set to NO_LOOP_CHANGE, the sounds current loop setting will be used.

Returns
ISMSound — The sound object that was affected. Returns null if the soundID was not found in the list.

See also
