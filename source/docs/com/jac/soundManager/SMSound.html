Constructor detail
SMSound()constructor
public function SMSound(soundID:String, className:String, autoLoop:Boolean = false)

Creates a new SMSound object and sets all default settings (_id, _state, _volume, _isPaused, _isStopped, _isPlaying, _autoLoop, _isMuted, _soundTransform)

Parameters
soundID:String — a unique ID that is used to reference the sound if managed by a SoundController.
 
className:String — the name of the export identifier of the sound in the Flash IDE library
 
autoLoop:Boolean (default = false) — sets infinite looping on or off. (this is a default setting, that can be changed at any point after creation.)

See also

playSound()
Method detail
checkMute()method
public function checkMute():void

Determines if the sound should be muted. This takes into account a codSoundController if there is on, as well as the SoundManager if there is one. This will do nothing if the sound is stopped.

This was designed to be called from itself, or a coSoundController. Chances are pretty good it won't need to be called by hand.

checkVolume()method 
public function checkVolume():void

Physically sets the volume of the flash sound. This takes into account any SoundControllers or SoundManagers.

Dispatches a SMSoundVolumeEvent.VOLUME_CHANGE event

This was designed to be called from itself, or a SoundController. Chances are pretty good it won't need to be called by hand.

forceVolume()method 
protected function forceVolume(newVol:Number):void

This will force the volume of this sound to be set to newVol. Doing this will disregard all other volume limits, including any SoundControllers or SoundManagers. Changes are pretty good that setVolume() should be used instead.

Parameters
newVol:Number — the desired volume of the sound. This will be clamped to a 0-1 range.

See also

setVolume()
handleSoundComplete()method 
protected function handleSoundComplete(e:Event):void

This is the event handler for when a sound finishes its current loop. If the sound has more loops to go, it is stopped and playSound() is called to start it again.

Parameters
e:Event — the event dispatched from the flash Sound class. (SOUND_COMPLETE)

See also

playSound()

playSound()method 
public function playSound(volume:Number, startTimeInMS:Number = 0, loops:int, currentLoop:int = 0):void

Plays the sound according to the passed in parameters.

Dispatches a SMSoundEvent.PLAY event

Parameters
volume:Number — the volume to set the sound to before playing. To leave the current volume pass in NO_VOL_CHANGE.
 
startTimeInMS:Number (default = 0) — how far in from the beginning of the sound to start playing, specified in milliseconds.
 
loops:int — how many times to loop the sound over when it finishes. To set to infinite pass in AUTO_LOOP. To leave the current settings pass in NO_LOOP_CHANGE.
 
currentLoop:int (default = 0) — what loop count to start at. Set to NO_CURRENT_LOOP_CHANGE to leave the current loop count as it is.
remove()method 
public function remove():void

Handles all sound cleanup before its removed from a SoundController. Additionally it stops the sound, and sets its volume to 0.

resumeSound()method 
public function resumeSound():void

Resumes a previously paused sound. This will do nothing if the sound was not paused, or is stopped.

stopSound()method 
public function stopSound(resetLoops:Boolean = true):void

Stops the current sound, if _isStopped is true.

Dispatches a SMSoundEvent.STOP event

Parameters
resetLoops:Boolean (default = true) — If TRUE, it will reset the played loop count to 0. If FALSE, it leaves the current loop count as is.
