Packagecom.jac.soundManager
Classpublic class SoundManager
InheritanceSoundManager flash.events.EventDispatcher

This object is used to manage a list of SoundController objects. Methods called on the SoundManager will be applied to all of the contained SoundControllers. There are three key objects to use this manager
  • SoundManager - the "global" controller that calls methods on its list of SoundControllers.
  • SoundController - A controller for a group of SMSound objects. The SoundController is used to manipulate individual SMSounds as well as groups of SMSounds.
  • SMSound - The actual sound object. This handles managing the sound, soundTransforms, soundChannels, and all other low level flash sound behaviours. This should (but isn't required to) be controlled from a SoundController

All of these objects can work together or individually. A SMSound can be created and used on its own. Likewise the SoundController does not need the SoundManager to function. To add/remove sound controllers please see addSoundController and removeSoundControllerByID

The SoundManager class is of the Singleton pattern.

To get a reference to it, please use

var _soundManager:SoundManager = SoundManager.getInstance();

To create more than one sound manager, simply extends this class and create a new singleton.

fadeVolume()method 
public function fadeVolume(endPercent:Number, duration:Number, curve:* = null):void

This will fade the volume of the SoundManager over a given time. This will also stop any tweens from a previous fadeVolume() on this SoundManager before the new fade is started.

Parameters
endPercent:Number — The final volume the SoundManger should reach.
 
duration:Number — The amount of time in seconds the fade should take to finish.
 
curve:* (default = null) — A TweenLite/Max curve definition to tween the volume by.
forceVolume()method 
public function forceVolume(vol:Number):void

This will force the volume of all managed SoundControllers and SMSounds to the passed in volume. Please note that this will override any other volume modifications currently applied to the objects. This will also stop any tweens from a fadeVolume() on this SoundManager before the volume is set.

Chances are pretty good that you want to use setVolume() in most cases.

Dispatches a SMSoundVolumeEvent.VOLUME_CHANGE event.

Parameters
vol:Number — A volume to force all sounds to, 0-1. This number is not clamped at this level, however it may be clamped further down the chain.

setFadeVol()method 
protected function setFadeVol(vol:Number):void

This is called on every update during a fadeVolume() call. It is used to notify the SoundControllers to check their volume.

You should not need to use this method directly.

Parameters
vol:Number — the current volume of the fade.
setVolume()method 
public function setVolume(vol:Number):void

This will set the volume of the SoundManager. Lowering this 50% will essentially lower each SoundController 50% from their current volume, which in turn lowers the SMSound 50% of its current volume. This will also stop any tweens from a fadeVolume() on this SoundManager before the volume is set.

Dispatches a SMSoundVolumeEvent.VOLUME_CHANGE event.

Parameters
vol:Number — A number from 0 - 1 that represents the new volume of the SoundManager. This number will be clamped to the 0-1 range. If 1.5 is passed, the volume will be set to 1.

See also

fadeVolume()
