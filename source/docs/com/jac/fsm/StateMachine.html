Classpublic class StateMachine
InheritanceStateMachine flash.events.EventDispatcher

A Class that handles the actual state switching. It deals with two StateControllers, the previous state, and the next state. It calls disable and exit on the previous state, then disable, enter, and enable on the next state. Each state is responsible for notifiying the state machine that it has completed, entering, exiting, enabling and disabling.

A StateMachine execution of changeState looks like this:

  • Disable Previous StateController if such state exists
  • Disable Next State StateController (before state is switched)
  • Wait for the previous StateController to report all of its sub objects have been disabled.
  • Wait for the next state StateController to report all of its sub objects have been disabled.
  • Exit Previous State
  • Wait for previous state to report all of its sub objects have finished exiting.
  • Enter Next State
  • Wait for next state to report all of its sub objects have finished entering.
  • Enable next state.
  • Wait for next state to report all of its sub objects have finished enabling.

changeState()method 
public function changeState(newState:StateController, blendTransitions:Boolean = false):Boolean

Switches the currentState to the newState. The operational flow is as follows:

  • Disable Previous StateController if such state exists
  • Disable Next State StateController (before state is switched)
  • Wait for the previous StateController to report all of its sub objects have been disabled.
  • Wait for the next state StateController to report all of its sub objects have been disabled.
  • Exit Previous State
  • Wait for previous state to report all of its sub objects have finished exiting.
  • Enter Next State
  • Wait for next state to report all of its sub objects have finished entering.
  • Wait for next state to report all of its sub objects have finished enter.
  • Enable next state.

Dispatches a StateChangeEvent.CHANGE event just before the change starts

Parameters
newState:StateController — A StateController object to enter into after the current state has exited.
 
blendTransitions:Boolean (default = false) — if set to true the current state will be told to exit(), and the next state will be told to enter() at the same time.

Returns
Boolean — True is returned if the change was started (!newState || (_inTransition && _blockDuringTransitions)) resulted in true. Returns False if the change wasn't started because of (!newState || (_inTransition && _blockDuringTransitions)) resulted in false.
clearStates()method 
public function clearStates():void

Sets _previousState, _nextState, _currentState to null. This should not really be needed, but is left in for convenience.

disableStates()method 
protected function disableStates():void

Calls disable on the current StateController during a state switch. Execution continues after a StateEvent.DISABLE_COMPLETE event is caught. This is an intermediate step in a state switch and can be overriden if needed when being subclassed.

enableStates()method 
protected function enableStates():void

Calls enable on the current StateController during a state switch. Execution continues after a StateEvent.ENABLE_COMPLETE event is caught at changeCompleted(). This is an intermediate step in a state switch and can be overriden if needed when being subclassed.

public function update():void

Calls the update function on the currentState. This can be used to run a "tick" system such as for a game loop. Each state could be notified of the "tick" and update as necessary for that particular frame.
